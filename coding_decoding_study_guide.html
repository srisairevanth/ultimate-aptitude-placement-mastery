Concept 1: Letter Shift (Forward/Backward)

Definition: Each letter is shifted by a fixed number of positions in the alphabet. Also called Caesar cipher. Intuition: Think of a wheel вҖ” rotating the alphabet by k positions. If k=3, AвҶ’D, BвҶ’E, etc. Formula: 
Forward shift by k: New = (Old + k) mod 26
Backward shift by k: New = (Old - k) mod 26
Alphabetically forward: AвҶ’BвҶ’CвҶ’...вҶ’Z
Shortcut вҖ” The Modulo Rule: 
After Z comes A (26 + 1 = 27 вүЎ 1 mod 26)
E + 3 = H (8 + 3 = 11)
Z + 2 = B (26 + 2 = 28 вүЎ 2 mod 26)
Example: "ABC" coded as "DEF" вҶ’ shift of +3. So "XYZ" вҶ’ "ABC" (wraps around). Common Mistakes:
  • вқҢ Forgetting to wrap around after Z
  • вқҢ Mixing up forward and backward shift
  • вқҢ Wrong modulo calculation

Trap 1: Wrong Wrap-Around After Z

  • Mistake: Z + 1 = A (correct) but Z + 5 = ? Counting wrong
  • Fix: Use formula: (P + k - 1) mod 26 + 1. Or: P + k; if >26, subtract 26.

Trap 2: A=0 vs A=1

  • Mistake: Using A=0, B=1 for coding (programmer's habit)
  • Fix: Standard: A=1, Z=26. Verify from the problem's examples.

Trap 3: Only Checking First Letter of Word

  • Mistake: Assuming a single-letter shift applies to the whole word without checking
  • Fix: Always verify on 2вҖ“3 letters before applying to all.

Trap 4: Missing Number Series Pattern

  • Mistake: Only checking first difference when it's actually a second-difference series
  • Fix: If dвӮҒ isn't constant, compute dвӮӮ. If dвӮӮ is constant вҶ’ quadratic series.

  • Mistake: A1, B4, C9, D16 вҶ’ only looking at letters (+1 each) or only numbers (1,4,9,16)
  • Fix: Always separate letter and number patterns. Letters: +1. Numbers: nВІ.
